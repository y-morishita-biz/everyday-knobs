# everyday knobs（1日1ノブ）要件定義書 v1

## 1. 背景・目的

ロータリーエンコーダ（EC11系・φ6軸）に挿さるツマミ（ノブ）を、ブラウザ上でパラメトリックに生成・プレビュー・エクスポートできるジェネレーターを作る。トグルとスライダーで全パラメータを自由にいじり、リアルタイムに3Dプレビューが更新され、STL / STEPで書き出せる。

「1日1ノブ」として、ジェネレーターの機能を1日1つ足しながら、その日の機能で作例ノブを1つ出す。機能追加と作例制作を両輪で回し、ジェネレーター自体を育てていく。

まず自分 / dotting dots 用の制作ツールとしてスタートするが、後でそのまま公開アプリに移行できる構造（サーバー不要・クライアントオンリー）で作る。

## 2. ゴール / 非ゴール

### ゴール
- EC11のφ6軸3種（セレーション軸・平軸・溝軸）に対応した軸穴を、トグルで切り替えてパラメトリック生成できる
- 基本本体形状を全パラメータ自由に調整でき、変更がリアルタイムにプレビューへ反映される
- STL と STEP の両方でエクスポートできる
- 毎日1機能ずつ足していける、要素分解された拡張しやすいアーキテクチャ
- 将来の公開（静的ホスティング）を犠牲にしない設計

### 非ゴール（v1ではやらない）
- **3Dプリントの嵌合・公差の作り込み**：v1では見た目とプレビュー体験を優先。実際に挿して使うための嵌合クリアランス・最小肉厚の最適化は後の日に足す（※アーキテクチャ上は「クリアランス」パラメータの差し込み口だけ用意しておく）
- **公開・共有機能**：マルチユーザー、ギャラリー、URL共有などは後フェーズ
- **EC11以外の軸規格**：他社軸・他径（φ6.35インチ系等）はP2。ただし軸モジュールを差し替え可能に作る
- **アセンブリ / 複数部品**：v1は単一ノブのみ

## 3. 技術スタック

| 領域 | 採用 | 理由 |
|---|---|---|
| 言語 | TypeScript | Webネイティブ、Claude Codeで保守しやすい |
| CADカーネル | **Replicad**（opencascade.js / WASM） | B-repソリッドを扱え、**STEPを本物の形式で出力可能**。フィレット・面取り・ブーリアンがカーネルレベルで揃う |
| 3D表示 | Three.js（Replicadのthreejsヘルパー経由） | プレビュー描画 |
| UI | React | トグル・スライダーのパラメータパネル |
| 計算分離 | WebWorker | カーネル計算をUIスレッドから分離し、操作中もプレビューを固まらせない |
| ビルド | Vite | WASM/Worker構成と相性が良い |
| エクスポート | STL（メッシュ） / STEP（B-rep） | Replicad標準機能 |
| 将来ホスティング | 静的（GitHub Pages / Vercel等） | サーバー不要 |

**補足**：なぜReplicadか。STEPは「B-rep（本物のソリッド）」フォーマットで、メッシュ系（Three.js直書き・JSCAD等）からは綺麗に出せない。STEPを扱うにはOpenCASCADEカーネルが必須で、それをブラウザ・サーバーレスで使える唯一に近い選択肢がReplicad。

## 4. 対象軸仕様（EC11 / ALPS）

軸はいずれも **φ6**。3種を軸穴（ネガ形状）としてモデル化し、ブーリアンで本体から引く。同じ仕組みで3種すべて実装できる。

1. **セレーション軸**：周状のローレット（スプライン）。歯数・モジュールをパラメータ化（一般的な定番は20山だが**要確定**）
2. **平軸（Dシャフト）**：φ6の片面をカット。カット深さ（残り厚み）・フラット長をパラメータ化
3. **溝軸**：軸先端のスロット。溝幅・溝深さをパラメータ化

> **寸法確定の方法**：ALPS製品ページ下部で配布されているSTEP（3D CAD）データから正確な寸法（歯数・カット深さ・溝寸法・有効軸長）を取得する。あわせてEC11実物のノギス実測と突き合わせる。嵌合クリアランスは独立パラメータとして外に出す。

## 5. 機能要件

### Must-Have（P0：これが無いとv1として成立しない）

- **[P0] 軸穴3種の切り替え生成**
  - Given: ユーザーが軸タイプをトグルで選ぶ
  - When: セレーション / 平軸 / 溝軸 を切り替える
  - Then: 対応する軸穴形状が本体から引かれ、プレビューが更新される
- **[P0] 基本本体形状のパラメトリック生成**：円柱ベース（径・高さが最小パラメータ）。全パラメータを自由に調整可能
- **[P0] リアルタイムプレビュー**：スライダー操作に追従して3Dプレビューが更新（WebWorkerで非同期計算）
- **[P0] STL / STEP エクスポート**：現在のパラメータでファイルを書き出せる
- **[P0] パラメータのバリデーション**：軸穴径が本体径を超える、肉厚がゼロ以下になる等の破綻形状を弾く（スライダー可動域＝破綻しない範囲）

### Nice-to-Have（P1：すぐ足したい高優先）

- **[P1] パラメータ設定の保存・再現**（JSONエクスポート / インポート）。作例の設定を残すために必要
- **[P1] 表面処理1種**（ローレット等）
- **[P1] 天面インジケーター**（ポインター / ノッチ）
- **[P1] 嵌合クリアランス・パラメータ**（印刷実用性の第一歩）

### Future Considerations（P2：v1では作らないが設計で潰さない）

- **[P2] 作例ギャラリー**（保存したJSON設定の一覧・呼び出し）
- **[P2] URL共有**（パラメータをURLに埋め込んで再現）
- **[P2] 軸規格の追加**（他径・他社軸）。軸モジュールを差し替え可能に設計しておく
- **[P2] 多様な本体形状**（円錐台・樽型・多角形）
- **[P2] 印刷品質の作り込み**（最小肉厚チェック・サポート最小化形状）

## 6. MVP（Day 1）受け入れ基準

最小スコープ。これが動けば「土台ができた」とみなす。

- [ ] Replicad + Three.js + React + WebWorker が起動し、ブラウザでプレビューが表示される
- [ ] 軸タイプをトグルで3種切り替えると、軸穴形状が切り替わる
- [ ] 円柱本体の径・高さをスライダーで変えると、プレビューがリアルタイム更新される
- [ ] 破綻パラメータ（軸穴 ≥ 本体径 等）がスライダー可動域で防がれる
- [ ] STL と STEP を書き出せ、外部CAD / スライサーで開ける

## 7.「1日1ノブ」ロードマップ（機能追加の種）

各日に1機能を足し、その機能で作例を1つ出す。要素分解された単位なので独立して足せる。

- 軸モジュールの公差パラメータ化
- 本体形状の追加（円錐台 → 樽型 → 多角形）
- 表面処理（ローレット → フルート → ディンプル → 刻み）
- 天面（ドーム → 凹み → ポインター → ノッチ）
- スカート / フランジ
- フィレット / 面取りの一括適用
- パラメータ設定のJSON保存・呼び出し
- 嵌合クリアランス → 実印刷で挿さる検証
- （以降）公開・ギャラリー・URL共有

## 8. 未決事項（Open Questions）

- **[要確定 / 実測] 軸の正確な寸法**：セレーションの歯数・モジュール、平軸のカット深さ、溝軸の溝寸法、各軸の有効長。ALPS配布STEP＋実測で確定する
- **[要決定] 本体形状の初期プリミティブ**：Day1は円柱のみで良いか、最初から円錐台まで入れるか
- **[要決定] バリデーションの方式**：スライダー可動域を動的に絞る方式か、入力後に警告を出す方式か（UX設計）
- **[後日] 印刷素材の前提**：嵌合クリアランスを詰める段階で、想定プリンタ・素材（PLA / PETG / レジン等）を決める
- **[後日] 公開時のライセンス**：生成物・ジェネレーター本体のライセンス方針
