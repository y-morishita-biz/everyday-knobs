# まいにちのぶ 開発ログ（#まいにちのぶ）

「1日1ノブ」の活動ログ。1日 = 1機能実装 + 1作例ノブ + 1 X（旧Twitter）投稿（**#まいにちのぶ**）。
新しい機能を実装し終えたら、このファイルの先頭に新しい Day エントリを **追記**する（運用手順は
[`CLAUDE.md`](../CLAUDE.md) を参照）。

エントリの雛形：

```markdown
## Day NN — YYYY-MM-DD — <ひとことタイトル>

- **実装した機能**：何を足したか（ジェネレーター側の機能）
- **作例ノブ**：その機能で作った1つのノブ（対象軸・寸法・狙い）
- **検証**：どう動作確認したか（build / 実機 / スクショ）
- **Xポスト下書き**：`#まいにちのぶ Day NN …`（投稿そのまま）
- **ブランチ / コミット**：`feat/day-NN-<slug>` / 主要コミットhash
```

## Day 26 — 2026-06-13 — Neumorphism UI ＆ ライト/ダークテーマ

- **テーマ**：世界観の刷新。全体を **Neumorphism（ソフトUI）** に作り替え、**light/dark テーマ
  切替**に対応（参考：Figma community の Neumorphism UI kit）。
  - **テーマ基盤**：CSS変数を `:root[data-theme="dark"|"light"]` の2系統に再編。初回フラッシュ
    防止のため index.html の**インラインスクリプトで先行設定**（localStorage→OSの`prefers-color-scheme`）。
    `useTheme` フック＋ヘッダの☀/🌙トグルで切替・永続化
  - **Neumorphism スキン**：ハードな枠線を廃し、`--neu-raised / -sm / inset` の二重シャドウ
    （左上ハイライト＋右下シャドウ）でカード・ボタン・トグル・プリセット・スライダー溝＆つまみ・
    ステータスを“面から押し出し/押し込み”表現に。選択・押下状態は inset
  - **3Dビューワも追従**：背景・グリッド色を CSS変数（`--viewer-bg`/`--grid-*`）から取得し、
    テーマ変更時に再着色
- **追補（Day26続き）：アクセントカラー・ピッカー**
  - プライマリ（アクセント）カラーを任意の色に変更可能に。8色のスウォッチ＋ネイティブ
    カラーピッカー（カスタム）＋「テーマ既定に戻す」。`useAccent` が `--accent` と
    コントラスト計算した `--accent-text` を上書き、localStorage 永続・初回はインライン先行設定
  - **UIだけでなく3Dノブの色も `--accent` 追従**（選んだ色で世界観が統一）
  - 外観設定を **⚙「外観」ポップオーバーに集約**（テーマのダーク/ライト＋カラー）。
    トップバーの単独テーマトグルから移設、外側クリック/Escで閉じる
  - timing修正：accentは effect でなく **setAccent内で CSS変数を同期更新**してから setState
    （子=Viewerの effectが親より先に走っても最新色を読める。テーマ修正と同じ対処）
- **不具合と修正**：トグル直後、Viewer の effect が App より先に走り `data-theme` 未反映の
  CSS変数を読んでビューワ背景だけ旧テーマのまま → `useTheme` の toggle で **DOM属性を同期的に
  更新**してから setState する方式に変更し解消（描画キャンバスのテーマ追従を確認）。
- **検証**：`npm run build` 通過。実機で dark⇄light 切替・**ビューワ背景/グリッドの追従**
  （canvasピクセル変化）・neumorphic shadow 適用・リロード後のテーマ永続を確認、エラー 0。
- **Xポスト下書き**：
  > #まいにちのぶ Day26 🎛️
  > UIをまるごとNeumorphism（ソフトUI）に刷新＋ライト/ダーク切替。さらにアクセントカラーを
  > 好きな色に変えられるピッカーも。選んだ色はUIも3Dノブも一括で着替え。枠線をやめて柔らかい
  > 影の"面"で、押すとへこむ手触りに。道具を作る道具にも、ちゃんと世界観を。

## Day 12 — 2026-06-11 — 性能改善DAY（操作レイテンシ 14〜22秒 → 0.5秒）

- **テーマ**：機能追加でなく**体感速度の最適化**。重いテクスチャ（Day 10 ヘリカル/ダイヤ）
  で「1操作ごとに十数秒固まる」問題を解消する。
- **やったこと**：
  1. **ビルド・スケジューラ（コアレッシング）** … `src/cad/useKnobMesh.ts` を新設。
     同時に走るビルドは常に1本、実行中の新規変更は**1枠の pending を上書き**するだけ。
     ドラッグで古い重ビルドが**N本積み上がる問題を根絶**（以前は5回動かせば5×14秒）。
     完了時に最新 pending を拾い直す“実質キャンセル”。同一パラメータは builtKey で再計算スキップ。
  2. **2段ビルド（下書き→仕上げ）** … テクスチャ有効時、操作中は**側面テクスチャを省いた
     滑面の下書き＋粗メッシュ**を即時表示。手を止めて 350ms 後に**テクスチャ込みの本ビルド**を
     1回だけ実行。worker の `build(params, draft)` に draft 経路を追加。
