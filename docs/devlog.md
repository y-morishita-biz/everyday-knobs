# まいにちのぶ 開発ログ（#まいにちのぶ）

「1日1ノブ」の活動ログ。1日 = 1機能実装 + 1作例ノブ + 1 X（旧Twitter）投稿（**#まいにちのぶ**）。
新しい機能を実装し終えたら、このファイルの先頭に新しい Day エントリを **追記**する（運用手順は
[`CLAUDE.md`](../CLAUDE.md) を参照）。

エントリの雛形：

```markdown
## Day NN — YYYY-MM-DD — <ひとことタイトル>

- **実装した機能**：何を足したか（ジェネレーター側の機能）
- **作例ノブ**：その機能で作った1つのノブ（対象軸・寸法・狙い）
- **検証**：どう動作確認したか（build / 実機 / スクショ）
- **Xポスト下書き**：`#まいにちのぶ Day NN …`（投稿そのまま）
- **ブランチ / コミット**：`feat/day-NN-<slug>` / 主要コミットhash
```

## Day 3 — 2026-06-11 — 円錐台ボディ（テーパー）

- **実装した機能**：本体形状をテーパー対応に（デザインサンプル分析の優先順②）。
  - 本体径を **下部 / 上部**の2スライダーに分離。上部 < 下部で上窄まりの円錐台、
    上部 > 下部で裾すぼまり（カップ型）。両径が等しいときは従来どおり真円柱
  - 実装：2円を `loftWith({ruled:true})` でロフトして角錐台ソリッドを生成。
    等径時は実績ある `makeCylinder` 経路にフォールバック
  - 天面エッジ処理（Day 2）の上限計算を**上部径基準**に修正（チャンファ/フィレットは
    天面リムに掛かるため）。テーパーと天面処理が破綻なく共存
- **作例ノブ**：下部φ20・上部φ10・H16・チャンファ1.5mm の円錐台（サンプル b0b69b48 の
  方向）。裾広がりカップ（下部φ10・上部φ28）も同パラメータで出せる
- **検証**：`npm run build` 通過。ヘッドレス **19ケース全パス**（両軸 × 円柱/上窄まり/
  裾広がり × 3エッジ＋極小上部径×フィレット最大）、STEP 出力も正常。実機ブラウザで
  上窄まり・裾広がりを目視確認、コンソールエラー 0
- **Xポスト下書き**：
  > #まいにちのぶ Day3 🎛️
  > 本体をテーパー対応に。上部径と下部径を別々に動かせるようにしたら、円錐台にも
  > 裾広がりカップにも化ける。Day2の天面チャンファとも自然に共存。
  > 2つの円をロフトするだけで、ツマミの佇まいが一気に増える。
- **ブランチ / コミット**：`claude/bold-mccarthy-j97rug`（継続作業）

## Day 2 — 2026-06-11 — 天面エッジ処理（チャンファ / フィレット）

- **実装した機能**：天面リムのエッジ処理を追加（デザインサンプル分析の優先順①）。
  - トグル3択：**なし / 面取り（チャンファ） / 丸め（フィレット）**
  - サイズスライダー（面取り幅 / 丸め半径、0.2mm〜動的上限）
  - 上限は**軸穴に到達しない半径方向マージン**と**本体高さの45%**から動的算出
    （`maxTopEdgeSize`）。本体径・軸タイプを変えてもスライダー可動域が破綻しない
  - 実装はエッジフィルタ（`e.inPlane("XY", bodyHeight)`）で天面リムだけに適用 →
    軸穴ブーリアンの前にかけることで対象エッジが常に一意
- **作例ノブ**：φ20 × H16・EC11 Dカット軸・**チャンファ 1.5mm**。
  サンプル画像のミニマル調理器ノブ（大ベベル金属調）の方向。フィレット最大(6.6mm)に
  振るとドーム型キャップ風に化けるのも確認
- **検証**：`npm run build` 通過。ヘッドレスで11ケース（両軸 × 3スタイル × 最大値
  ＋極小ボディφ10×H6）全部 OK、STEP 出力も正常。実機ブラウザでチャンファ/フィレット/
  なしの切替・スライダー上限・「なし」時のスライダー非表示を目視確認、コンソールエラー 0
- **Xポスト下書き**：
  > #まいにちのぶ Day2 🎛️
  > 天面エッジ処理を追加。面取り(チャンファ)/丸め(フィレット)をトグルで切替、
  > 幅・半径はスライダーで。たった1パラメータでツマミの表情が激変する。
  > チャンファ1.5mmでミニマル家電風、フィレット全開にするとドーム型に。
  > 軸穴に食い込まない範囲だけ動かせる安全設計。
- **ブランチ / コミット**：`claude/bold-mccarthy-j97rug`（継続作業）

## Day 1 — 2026-06-10 — MVP ジェネレーターが動いた

- **実装した機能**：ブラウザで動くパラメトリック・ノブ・ジェネレーターの MVP。
  - 本体（円柱）＋軸穴（ネガ形状）をブーリアンで生成
  - 軸タイプをトグルで切替：**EC1110120005（φ6 Dカット）** ⇄ **EC12E085（φ6 中空＝丸穴）**
  - スライダー：本体径 / 本体高さ / 軸穴深さ / 嵌合クリアランス → リアルタイムにプレビュー更新
  - 破綻形状（軸穴 ≥ 本体径、肉厚ゼロ等）をスライダー可動域（最小肉厚 1.5mm 確保）で防止
  - **STL / STEP エクスポート**
  - CAD カーネル（Replicad = OpenCASCADE WASM）を **WebWorker** で実行し UI を固めない。
    プレビューは Three.js。Vite + React + TypeScript。
- **作例ノブ**：円柱ノブ **φ20 × H16mm**、EC11 φ6 Dカット軸用、嵌合クリアランス 0.15mm。
  （ジェネレーターのデフォルト＝最初の1個）
- **検証**：
  - `npm run build`（型チェック＋バンドル）通過
  - ヘッドレス CAD で両軸タイプの boolean cut / STL / STEP 生成成功（STEP は `ISO-10303-21` 妥当）
  - 実機ブラウザ（Chromium）で目視確認：底面に **Dカットの軸穴**を確認、トグル・スライダーの
    リアルタイム更新、STL/STEP のダウンロード、コンソールエラー 0 件
- **Xポスト下書き**：
  > #まいにちのぶ Day1 🎛️
  > ブラウザで動くノブ・ジェネレーターのMVPができた。ALPS EC11(φ6 Dカット)とEC12E(中空)の
  > 軸穴をトグルで切替、本体径/高さ/クリアランスをスライダーで即プレビュー、STL/STEPで書き出し。
  > Replicad(WASM)+Three.js、サーバー不要のクライアントオンリー。
- **ブランチ / コミット**：（基盤期は `claude/bold-mccarthy-j97rug`） / `b3a5eb5`
