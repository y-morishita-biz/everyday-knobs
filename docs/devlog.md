# まいにちのぶ 開発ログ（#まいにちのぶ）

「1日1ノブ」の活動ログ。1日 = 1機能実装 + 1作例ノブ + 1 X（旧Twitter）投稿（**#まいにちのぶ**）。
新しい機能を実装し終えたら、このファイルの先頭に新しい Day エントリを **追記**する（運用手順は
[`CLAUDE.md`](../CLAUDE.md) を参照）。

エントリの雛形：

```markdown
## Day NN — YYYY-MM-DD — <ひとことタイトル>

- **実装した機能**：何を足したか（ジェネレーター側の機能）
- **作例ノブ**：その機能で作った1つのノブ（対象軸・寸法・狙い）
- **検証**：どう動作確認したか（build / 実機 / スクショ）
- **Xポスト下書き**：`#まいにちのぶ Day NN …`（投稿そのまま）
- **ブランチ / コミット**：`feat/day-NN-<slug>` / 主要コミットhash
```

## Day 10 — 2026-06-11 — 綾目・斜めローレット（ヘリカル / ダイヤ）

- **実装した機能**：側面テクスチャに **斜め（ヘリカル）** と **綾目（ダイヤ）** を追加
  （分析の優先順⑦）。Day 6 の縦溝を一般化。
  - カッター円柱を円周方向に **ねじれ角**ぶん傾けて斜め溝に。ダイヤは +θ / −θ の2方向を
    別々にカットして交差ハッチ
  - 本数 / 深さ / 太さ（Day 6・7）に加え **ねじれ角**スライダー（10〜30°）
  - チャンファ・天面凹み・指標・スカート・テーパー・両軸すべてと共存
- **作例ノブ**：φ20・H20・チャンファ＋刻線に **ダイヤ綾目（各24本・30°）**。サンプル
  c1a896ab / eee3bf9f の綾目ローレットつまみ
- **検証**：`npm run build` 通過。ヘッドレス 4ケース（ヘリカル/ダイヤ × 円柱/テーパー/
  両軸/チャンファ+スカート）全パス・STEP正常。実機ブラウザでヘリカル/ダイヤを目視確認、
  コンソールエラー 0
- **⚠ 性能と設計判断**：**斜め溝は OCCT(WASM) のスキュー・ブーリアンが重い**。
  ヘリカル ≈ 5〜8秒、ダイヤ ≈ 14秒（縦溝が約1秒なのに対し）。傾けたシリンダ ∩ 本体の
  交差が高コストで、ねじれ押し出し（twistAngle, 高速）も**それ自体は速いが、本体との
  fuse / cut / intersect が全てハング級に遅い**ことを実測で確認（複数方式を検証）。
  - 対応：**ブーリアン方式を採用**（正しく全機能と共存）。ねじれ角を10〜30°に制限、
    ダイヤは2方向を別カット、UIに「数秒かかります」を明示、WebWorker で UI は固めない
  - **将来の最適化Day の種**：重いテクスチャは「ライブは滑面プレビュー → 確定/書き出し時に
    knurl をベイク」する遅延適用モデルに分離する余地
- **Xポスト下書き**：
  > #まいにちのぶ Day10 🎛️
  > ローレットを斜め（ヘリカル）と綾目（ダイヤ）に拡張。カッターをねじって交差させると
  > あのダイヤ目の手触りに。CADカーネルのブーリアンが重くて数秒かかるけど、本物の
  > knurlがブラウザで生成できるのは気持ちいい。"滑り止め"も立派な造形。

## Day 7 — 2026-06-11 — スタイル微調整パラメータの開放

- **実装した機能**：これまで実装した各スタイルの**内部固定値をスライダーとして開放**。
  - **リセス/ディッシュ**：縁の幅（0.5mm〜、凹み半径が最小φ4を切らない範囲で動的上限）
  - **指標**：彫り深さ（0.4〜2.5mm、軸穴上の肉厚でクランプ）／**線の長さ・中心からの距離**
    （刻線とディンプルで共用の「リーチ」パラメータ、ラベルだけ切替）
  - **ローレット**：溝の太さ（ピッチ比 40〜120%。85%が従来値）。120%にすると溝同士が
    つながりウェーブ状、40%で細いスリット調と表情が大きく変わる
  - すべて `set()` 内の動的クランプに参加。99mm や 200% を投げても安全に丸まる
- **作例ノブ**：①縦溝12本×太さ120%×深さ1.5mm の太溝グリップ（ギアっぽい無骨系）
  ②リセス縁3mm＋刻線を彫り2.5mmでリム貫通させた二段天面
- **検証**：`npm run build` 通過。ヘッドレス **38ケース全パス**（縁幅/リーチ/彫り深さ/
  溝太さの全スイープ＋極端値クランプ＋全部入り両軸）。実機ブラウザで太溝・縁幅・
  短い刻線を目視確認、コンソールエラー 0
- **学び**：深いリセスに浅い刻線を組み合わせると刻線が「宙に消える」（天面基準で彫るため）。
  クランプは形状破綻を防ぐが、**見た目の整合**はパラメータ間の意味づけが必要。
  将来「リセス底基準で彫る」オプションの種
- **Xポスト下書き**：
  > #まいにちのぶ Day7 🎛️
  > 各スタイルの"中に隠れてた数字"をスライダーに開放。リセスの縁幅、指標の長さと彫り深さ、
  > ローレットの溝の太さ。同じ縦溝でも48本×細目はナール、12本×太め120%はギアの顔。
  > パラメトリックの楽しさはこの「同じ仕組み、別の表情」にある。
- **ブランチ / コミット**：`claude/bold-mccarthy-j97rug`（継続作業）

## Day 4 — 2026-06-11 — 天面リセス / ディッシュ

- **実装した機能**：天面の凹み加工を追加（デザインサンプル分析の優先順③）。
  - トグル3択：**フラット / リセス（一段凹み）/ ディッシュ（すり鉢）**
  - リセスは円柱ポケットを天面からカット、ディッシュは半径と深さから球半径を逆算
    （`ρ=(a²+h²)/2h`）して球をカット＝なめらかな凹面
  - 深さ上限は **軸穴の天端との間に MIN_WALL を残す**よう動的算出（`maxTopRecessDepth`
    = 本体高さ − 軸穴深さ − 1.5mm）。軸穴を浅くすると深い凹みにできる旨をヒント表示
  - 凹み半径は**天面エッジ処理後の平坦半径**（`flatTopRadius`）基準。チャンファ/フィレット・
    テーパーと破綻なく共存
- **作例ノブ**：φ20・H16・チャンファ1.5mm・軸穴9mm に **すり鉢5.5mm**。サンプル
  84792948（深い凹面天面）の方向。リセスにすると一段凹みのプッシュボタン風
- **検証**：`npm run build` 通過。ヘッドレス **54ケース全パス**（両軸 × フラット/リセス/
  ディッシュ × 円柱/上窄まり/裾広がり × 3エッジ）、STEP 正常。実機ブラウザでリセス/
  ディッシュを上面から目視確認、コンソールエラー 0。テスト harness を実ソース
  （`knob.ts` を esbuild バンドル）で回す方式に変更し、ロジック二重持ちを解消
- **Xポスト下書き**：
  > #まいにちのぶ Day4 🎛️
  > 天面に凹みを追加。一段凹みの「リセス」と、なめらかな「すり鉢（ディッシュ）」を
  > トグルで。すり鉢は半径と深さから球をフィットさせて生成。指の収まりが気持ちいい
  > 天面に。軸穴と干渉しない深さだけ掘れる安全設計。
- **ブランチ / コミット**：`claude/bold-mccarthy-j97rug`（継続作業）

## Day 2 — 2026-06-11 — 天面エッジ処理（チャンファ / フィレット）

- **実装した機能**：天面リムのエッジ処理を追加（デザインサンプル分析の優先順①）。
  - トグル3択：**なし / 面取り（チャンファ） / 丸め（フィレット）**
  - サイズスライダー（面取り幅 / 丸め半径、0.2mm〜動的上限）
  - 上限は**軸穴に到達しない半径方向マージン**と**本体高さの45%**から動的算出
    （`maxTopEdgeSize`）。本体径・軸タイプを変えてもスライダー可動域が破綻しない
  - 実装はエッジフィルタ（`e.inPlane("XY", bodyHeight)`）で天面リムだけに適用 →
    軸穴ブーリアンの前にかけることで対象エッジが常に一意
- **作例ノブ**：φ20 × H16・EC11 Dカット軸・**チャンファ 1.5mm**。
  サンプル画像のミニマル調理器ノブ（大ベベル金属調）の方向。フィレット最大(6.6mm)に
  振るとドーム型キャップ風に化けるのも確認
- **検証**：`npm run build` 通過。ヘッドレスで11ケース（両軸 × 3スタイル × 最大値
  ＋極小ボディφ10×H6）全部 OK、STEP 出力も正常。実機ブラウザでチャンファ/フィレット/
  なしの切替・スライダー上限・「なし」時のスライダー非表示を目視確認、コンソールエラー 0
- **Xポスト下書き**：
  > #まいにちのぶ Day2 🎛️
  > 天面エッジ処理を追加。面取り(チャンファ)/丸め(フィレット)をトグルで切替、
