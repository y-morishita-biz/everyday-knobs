# everyday knobs — 公開サイト（LP＋ジェネレーター）要件定義

> このドキュメントは、GitHub Pages で公開する **紹介LP＋マニュアル＋ジェネレーター** の
> 構成・内容・デザイン要件を定義したもの。ビジュアルデザイン検討（Claude Design 等）の
> インプットとして使う。**コピーは「方向性／ドラフト」**であり、デザイン段階で調整可。
> 実装は本要件確定後に着手する。

- ステータス：**要件定義（デザイン検討用ドラフト）**
- 対象成果物：公開LP（`/`）／マニュアル（`/manual` または LP内）／ジェネレーター（`/app/`、実装済み）
- 関連：[`README.md`](../README.md) / [`docs/devlog.md`](devlog.md) / [`posts/まいにちのぶ.md`](../posts/まいにちのぶ.md)

## 0. このプロダクトは何か（前提共有）

**everyday knobs（1日1ノブ）** は、ロータリーエンコーダ（ALPS **EC11** φ6 Dカット軸 /
**EC12E** φ6 中空軸）に挿さるツマミを、**ブラウザ上でパラメトリックに生成 → 3Dプレビュー →
STL/STEP 書き出し** できるジェネレーター。インストール不要・サーバー不要のクライアントオンリー。

「**1日1ノブ**」— 30日間、1日1機能ずつ実装し、その日の機能で作例を1つ出し、X（**#まいにちのぶ**）に
1投稿する取り組みで育てた。**開発の物語そのものがコンテンツ**になっている点を活かす。

## 1. 目的・ゴール / 非ゴール

### LPの目的
1. **「触ってもらう」**：訪問者を最短でジェネレーター（`/app/`）に送る（主要CV = アプリ起動）
2. **「何ができるか」を一目で伝える**：形状バリエ・印刷実用・共有のしやすさ
3. **作者と#まいにちのぶの世界観を伝える**：30日の物語、X(@penomo)への導線（副次CV = Xフォロー）

### ゴール
- ファーストビューで「ブラウザでツマミを作ってSTL/STEPで出せる」が即わかる
- スマホでも崩れず、CTAが常に届く範囲にある
- マニュアルで「使い方」「印刷で実際に挿す方法」まで自己完結
- アプリ本体のデザイン世界観（Neumorphism＋テーマ＋アクセント色）と地続き

### 非ゴール
- 会員登録・課金・問い合わせフォーム等の動的機能（静的サイトのため）
- ブログ的な記事量産（#まいにちのぶは別途アーカイブを参照リンク）

## 2. ターゲット & トーン

### 想定ユーザー
- 自作キーボード／シンセ／エフェクター／モジュラー／自作家電・DIYガジェットを作る人
- 3Dプリンタを持っていて「市販に無いツマミ」を作りたい人
- 個人メーカー／同人ハード勢（dotting dots 的な制作者）
- 「パラメトリックで形をいじる」こと自体が楽しい層

### トーン＆マナー（言葉の温度）
- **道具らしく・誠実・少し職人気質**。誇張より「実際に挿さる／実際に作れる」を見せる
- 専門用語（Dカット・公差・ローレット）は使うが、初見にも1行で噛み砕く
- 既存の#まいにちのぶ投稿の語り口（「"ツマミ"な指掛かり」「据わりのいい」等）を踏襲

## 3. サイト構成（IA / サイトマップ / URL）

```
https://y-morishita-biz.github.io/everyday-knobs/        ← 公開ルート
├─ /            LP（トップ：紹介・ショーケース・導線）        ← 新規作成
├─ /manual      使い方マニュアル（操作・パラメータ・印刷）       ← 新規作成（LP内アンカーでも可）
└─ /app/        ジェネレーター本体（React+WASM）             ← 実装済み（app/dist を配置）
```

- **`/manual` は独立ページ or LP内の長セクション**のどちらでも可。デザインで選択
  （情報量が多いので独立ページ推奨。LPからは「使い方を見る」で遷移）
- ルーティングはハッシュ不使用の素の静的パス。`/app/` への遷移は通常リンク（重いWASMはLPでは読まない）
- グローバルナビ（LP/マニュアル共通）：ロゴ｜特徴｜作例｜使い方(マニュアル)｜#まいにちのぶ｜**［ジェネレーターを開く（主CTA）］**

## 4. デザイン要件（トンマナ・システム）

### 世界観
- **アプリ本体と同じ Neumorphism（ソフトUI）** を基調に。枠線レス＋二重シャドウ（左上ハイライト／右下シャドウ）、角丸、面の押し出し/押し込み
- **ライト/ダーク両対応**（LPも切替できると統一感◎。最低限どちらか1テーマで成立すればOK）
- **アクセントカラー**：アプリ既定の紫（dark `#9d7bff` / light `#7c5cff`）を基調。アプリ同様に
  「色を変えられる」ことが売りなので、LPのキービジュアルで複数アクセント色の作例を見せると強い

### カラー（アプリのCSS変数を流用）
| 役割 | Dark | Light |
|---|---|---|
| 背景/面 | `#262a31` | `#e6e9f0` |
| 文字 | `#d4d9e3` | `#3f4458` |
| ミュート文字 | `#8b92a3` | `#6d7286` |
| アクセント | `#9d7bff` | `#7c5cff` |
| 影(濃) / 影(淡) | `#1e2127` / `#30343d` | `#c5c9d6` / `#ffffff` |

### タイポ
- 既存：`system-ui`系。見出しは太め・字間広め（アプリの `panel__title` に準拠）。
  デザインで日本語見出しフォントを足すなら、本文可読性優先（明朝より角ゴシック推奨）

### モーション
- 控えめ・上質に。3Dノブの**自動回転（ターンテーブル）GIF/動画**はヒーローで効果大
- `prefers-reduced-motion` 配慮（アプリと同様、動きを止められること）

### レスポンシブ
- **モバイルファースト**。縦1カラム、CTAは追従 or 各セクション末に再掲
- ブレークポイントの目安：〜760px（モバイル）/ 〜1100px / それ以上

## 5. LP の要件（セクション定義）

> 各セクション：**目的 / 構成要素 / コピー方向性（ドラフト） / 使用アセット / CTA**。
> 並び順は上から。コピーは仮、デザインで磨く前提。

### S1. ヒーロー（最重要）
- **目的**：3秒で「何ができるか」＋即CTA
- **要素**：キャッチコピー／サブコピー／主CTA（ジェネレーターを開く）／副CTA（使い方を見る）／
  ヒーロービジュアル（回転する3Dノブ or 高解像度の作例）
- **コピー方向性（案）**：
  - 主見出し：「**ブラウザで、自分だけのツマミを。**」 / 別案「**回す道具を、パラメトリックに。**」
  - サブ：「ALPSエンコーダ用のノブを、その場でデザイン → STL / STEP で書き出し。インストール不要・全部ブラウザで完結。」
  - 主CTAボタン：**［ジェネレーターを開く →］**（`/app/`）
  - 副CTA：［使い方を見る］（`/manual`）
- **アセット**：`docs/img/hero.png`（バレルグリップ）／可能なら自動回転の短い動画 or GIF
- **メモ**：アクセント色違いのノブを3〜4個並べる「色変え」ビジュアルも候補

### S2. 価値（3つのポイント）
- **目的**：詳細機能の前に「なぜ良いか」を3点で
- **要素**：アイコン＋見出し＋1〜2行 ×3
- **コピー方向性（案）**：
  1. **インストール不要** — 「ブラウザを開けばすぐ。サーバーもアプリも要らない」
  2. **思いのままの形** — 「丸・多角形・波型・指針、ローレットも目盛りもドームも。組み合わせは無限」
  3. **実際に挿さる・刷れる** — 「実寸公差と公差テストピースで、3Dプリントが本当に軸に挿さる」

### S3. 機能ショーケース（ビジュアル主役）
- **目的**：形のバリエ・テクスチャ・印刷・共有を「見て」伝える
- **構成**：4〜5ブロックを大きめの画像＋短文で（横スクロール or 縦積み）。**強調する機能（おすすめ）**：
  | ブロック | 見せ方 | アセット |
  |---|---|---|
  | **多彩な本体形状** | 丸/六角/角丸スクエア/花形ロブ/チキンヘッド/樽/くびれ を並べる | `day11-hex` `day17-lobed` `day18-pointer` `day16-barrel` `day16-waist` |
  | **5種の側面テクスチャ** | 縦溝/ヘリカル/綾目ダイヤ/横溝/指スクープ | `day10-diamond` `day10-helical` `day19-rings` `day19-scallops` `day6-flutes24` |
  | **天面と指標・目盛り** | ドーム/すり鉢/リセス＋刻線/ダイヤル目盛り | `day20-dome` `day4-dish` `day15-dial` |
  | **印刷で実際に挿す** | 軸穴実寸表示＋公差テストピース | `day14-panel` |
  | **共有・再現** | URL共有/注文コード/プリセット/マイプリセット | `day25-mypresets` `day13-gallery` |
- **メモ**：各ブロックに「この形を試す」→該当プリセットの共有URL（`/app/#c=...`）を貼れると強い
  （※プリセットの共有コードは実装側から取得して後で埋め込み可能）

### S4. 使い方 3ステップ
- **目的**：「自分にもできそう」と思わせる
- **要素**：3カード（番号＋アイコン＋見出し＋1行）
  1. **選ぶ** — 作例ギャラリーから近い形をタップ
  2. **調整** — スライダー＆トグルで自分好みに（🎲ランダムでも）
  3. **書き出す** — STL / STEP / PNG。印刷して軸に挿す
- **CTA**：［詳しい使い方（マニュアル）］

### S5. 作例ギャラリー
- **目的**：表現の幅を一覧で。眺めて楽しい
- **要素**：プリセット16種のサムネ格子（アプリのSVGグリフ風 or 実レンダー）。クリックで `/app/` の該当設定へ
- **アセット**：`day*` レンダー群、または `/app/` のプリセット名一覧
  （ベーシック/ローレット円筒/綾目ダイヤ/テーパー・ディッシュ/フランジ台座/六角ボルト/角丸スクエア/
  ヘリカル・テーパー/ドームトップ/コインエッジ/指スクープ/チキンヘッド/花形ロブ/バレルグリップ/目盛りダイヤル）

### S6. 対象エンコーダ（信頼の担保）
- **目的**：「自分の部品で使えるか」を明確化
- **要素**：EC11（φ6 Dカット）/ EC12E（φ6 中空）の2機種カード＋一言（実寸ベース・公差は調整可）
- **コピー**：「ALPS EC1110120005（φ6 Dカット）と EC12E085（φ6 中空）に対応。軸寸法はメーカーSTEPから実測」

### S7. #まいにちのぶ（物語＆SNS導線）★必須
- **目的**：作り手の物語で愛着を作り、**X(@penomo) への導線**を張る
- **要素**：
  - 短いストーリー（「30日間、1日1機能ずつ。#まいにちのぶ で毎日投稿しながら育てた」）
  - **30日のハイライト**（タイムライン or サムネ帯：Day1→Day30 の代表数枚）
  - **X CTA**：「**@penomo をフォロー**」ボタン → `https://x.com/penomo`
  - ハッシュタグ **#まいにちのぶ** の明示（クリックでXの検索 or タグページへでも可）
- **アセット**：`posts/まいにちのぶ.md` の各Day画像、`day30-hero.png`

### S8. 最終CTA（クロージング）
- **目的**：離脱前にもう一度アプリへ
- **要素**：大きな見出し＋主CTA［ジェネレーターを開く →］＋サブ（GitHub/X）
- **コピー（案）**：「**さあ、回す道具を作ろう。**」

### フッター（全ページ共通）
- ロゴ／一言／リンク群：ジェネレーター（`/app/`）｜マニュアル｜GitHub（リポジトリ）｜**X @penomo**｜#まいにちのぶ
- クレジット：制作者（penomo / dotting dots）／技術（Replicad・Three.js・React）／ライセンス表記（※要決定）

## 6. 共通要素・横断要件

- **ヘッダ**：固定 or スクロール追従。右端に常時主CTA（ジェネレーターを開く）。テーマ切替（☀/🌙）があると統一感
- **OGP / SNSカード（重要：Xでの拡散前提）**：
  - `og:title` / `og:description` / **`og:image`（1200×630）** / `twitter:card = summary_large_image` / `twitter:site = @penomo`
  - OGP画像はヒーローの作例を流用 or 専用に1枚用意
- **ファビコン / アプリアイコン**：ノブのシルエット（🎛️ 的）。要新規作成
- **パフォーマンス**：LPは軽量（WASMを読み込まない）。画像は適切に圧縮、遅延読込
- **アクセシビリティ**：見出し階層・代替テキスト・コントラスト（特にライト）・フォーカス可視・reduced-motion
- **計測（任意）**：必要なら静的でも可能な軽量アクセス解析（要否・手段は要決定）

## 7. マニュアルの要件（`/manual`）

> 「初めて触る人が、印刷して軸に挿すところまで自己完結できる」ことがゴール。
> 章立て（TOC）と各章の中身を定義。図はアプリのスクショ（`posts/images/`）を流用。

### M0. はじめに
- これは何か（1段落）／動作環境（モダンブラウザ・初回WASM数秒）／対象エンコーダ2種

### M1. 基本の流れ（最短ルート）
- ①ギャラリーで近い作例を選ぶ → ②スライダー/トグルで調整 → ③STL/STEP書き出し → ④印刷して軸に挿す
- 図：`day13-gallery` → 調整 → `day14-panel`

### M2. 画面の見方
- 3エリア：**コントロールパネル**（左/下：カテゴリ別アコーディオン）／**3Dビューワ**（フィット・視点・自動回転・PNG）／**トップバー**（戻す/進む/ランダム/?/⚙）
- 図：`day21-accordion` `day23-toolbar`

### M3. パラメータ・リファレンス（早見表）
- **カテゴリごとの表**：各パラメータの「名称 / 役割 / 範囲・単位 / 形への影響・コツ」。最低限カバーすべき項目：
  - **軸・取り付け**：軸タイプ(EC11/EC12E)、軸穴深さ、嵌合クリアランス（±・実寸表示の読み方）
  - **本体形状・サイズ**：断面(丸/多角形/波型/指針)、角数・角丸、ロブ数・深さ、指針長さ、上下径、高さ、胴のふくらみ(樽/くびれ)
  - **天面**：エッジ(面取り/丸め＋サイズ)、スタイル(フラット/リセス/すり鉢/ドーム＋深さ・縁幅)
  - **側面**：テクスチャ(なし/縦溝/ヘリカル/ダイヤ/横溝/スクープ＋本数・深さ・太さ・ねじれ角)、スカート(フランジ＋径・高さ)
  - **指標・目盛り**：指標(刻線/ディンプル＋サイズ・角度・深さ・距離)、目盛り(本数・主目盛り間隔・角度範囲)
  - **印刷サポート**：底面の面取り、公差テストピース
- ＋「組み合わせのコツ」コラム（例：角丸多角形/波型/指針は天面の面取りが自動スキップ、等の仕様も明記）

### M4. 3Dプリント・ガイド（実際に挿す）★差別化ポイント
- **嵌合クリアランスの考え方**（プラスで緩く・マイナスで圧入、実寸表示の見方）
- **公差テストピースの使い方**：書き出し→印刷→5段を順に挿す→一番いい段の番号(刻み目)を読む→本体の値に反映
- 挿入リードイン面取り／底面エレファントフット対策の自動付与の説明
- **素材別の目安**（PLA/PETG等のクリアランス傾向）※一般論として。最終は実測推奨
- 図：`day14-panel`

### M5. 保存・共有・再現
- **URL共有**（リンクを送るだけで同じ形）／**JSON保存・読込**／**注文コード**（メーカーに送ってプリント依頼）／**マイプリセット**（ブラウザ保存）
- 図：`day25-mypresets` `day8-config`

### M6. キーボードショートカット
- 一覧表：`Ctrl/⌘+Z` 戻す｜`Ctrl/⌘+Shift+Z` 進む｜`R` ランダム｜`F` フィット｜`1/2/3` 視点｜`0` 自動回転｜`?` ヘルプ

### M7. よくある質問・トラブルシュート
- 「重い／数秒固まる」→ 綾目・横溝など重いテクスチャは静止後に仕上げ（2段ビルド）。仕様
- 「印刷したら挿さらない／きつい」→ クリアランス調整・テストピースへ
- 「対応してない軸は？」→ 現状EC11/EC12Eのみ（将来拡張の余地）
- 「商用利用は？データの権利は？」→ ライセンス方針（※要決定）

## 9. 技術・実装メモ（デザイン後の実装方針）

- **配信**：GitHub Pages（public）。`/` にLP、`/app/` にジェネレーター。`.nojekyll` を置く
- **デプロイ**：GitHub Actions で `app` をビルド→`/app/`、LP/manualを `/` に組み立てて公開（`main` push で自動）
- **LPの作り**：軽量な静的HTML/CSS（フレームワーク任意）。アプリのWASMはLPでは読み込まない
- **アプリ側**：`base: "./"`・`replicad_single`（特殊ヘッダ不要）で、サブパス配信に対応済み
- **整合性**：LPのテーマ/アクセント変数はアプリの値と一致させる（[セクション4](#4-デザイン要件トンマナシステム)の表）
